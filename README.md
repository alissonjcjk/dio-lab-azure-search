## 🎯 O que eu fiz e aprendi nesse laboratório?

O objetivo principal aqui foi entender na prática como funciona um motor de busca inteligente corporativo. Não é apenas aquele clássico "buscar por palavra exata", mas sim usar IA para enriquecer o processo. Dividi o que aprendi e meus *insights* em três grandes etapas:

### 1️⃣ Ingestão de Conteúdo para IA
Antes de fazer as consultas, o sistema precisa conseguir ler os dados originais. O mais legal foi ver a capacidade do Azure em extrair e se conectar a diferentes fontes de informações.
* **Brainstorm de Estudante:** Essa etapa me lembrou muito as matérias introdutórias de inteligência artificial e ciência de dados, nas quais vemos que o coração de um bom resultado é justamente um pré-processamento bem feito e de onde vem os nossos dados. Ter isso resolvido na "pipeline" do Azure é muito produtivo.

### 2️⃣ Criação de Índices Inteligentes (A Engine do Negócio)
Essa com certeza é a minha parte favorita. O **Indexador** mapeia as colunas/dados e o **Index** mantém as informações organizadas e prontas para pesquisa. O que dá o diferencial são as *Skills* de IA que podemos acoplar (ex: tirar texto de imagem via OCR, detectar idioma...).
* **O que mais impressionou:** Nas aulas de Estrutura de Dados a gente estuda a fundo algoritmos de busca, árvores e tabelas hash. Conseguir enxergar toda essa teoria pesada rodando "nos bastidores" de forma tão escalável no Cloud me ajudou a entender de vez por que as grandes empresas estão migrando rápido para serviços assim. 

### 3️⃣ Exploração Prática e Mineração de Conhecimento
No final, era hora de consultar. Usando a interface (UI) do *Search explorer*, pude rodar as queries. Os retornos dos resultados chegam bem detalhados (formato JSON), trazendo atributos adicionais e *scores* (pontuações de relevância). Fica muito claro que é possível ligar esse back-end diretamente com uma aplicação front-end que o usuário final ou analista de dados usaria na web.
